# Security Policy

## Supported Versions

Only the latest release line is actively supported for security fixes.

## Reporting a Vulnerability

Please do not open public issues for unpatched vulnerabilities.

Send a private report with:

- affected version
- reproduction steps
- impact assessment
- optional patch suggestion

Temporary contact: `security@example.com` (replace with your real mailbox before release).

## Sensitive Data Policy

- Do not commit real HAR captures containing tokens/cookies/personal data.
- Do not commit production credentials in scripts or docs.
- Use environment variables for all secrets.
